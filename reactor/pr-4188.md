## 🎓 What I Learned

### Technical Learning

#### 1. Reactive Programming Anti-patterns
**Fire-and-forget Problems:**
- Errors disappear silently
- No backpressure handling
- Resource leaks possible
- Chain breaks reactive composition

**Proper Patterns:**
```java
// Composition (preferred)
return flux
  .flatMap(this::process)
  .doOnNext(this::sideEffect);

// Imperative boundary (acceptable)
flux.subscribe(
  value -> {},
  error -> log.error("Failed", error),
  () -> {}
);
```

#### 2. Reactor Documentation Structure
- Uses clear WARNING blocks
- Provides anti-pattern vs preferred examples
- Focuses on "why" not just "how"
- Links to related concepts
