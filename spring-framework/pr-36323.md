## 🎓 What I Learned

### Technical Learning

#### 1. AsciiDoc Tab Syntax
```asciidoc
[tabs]
======
Java::
+
[source,java,indent=0,subs="verbatim,quotes"]
----
// Java code
----

XML::
+
[source,xml,indent=0,subs="verbatim,quotes"]
----
<!-- XML code -->
----
======
```

#### 2. Spring Documentation Structure
- `framework-docs/modules/ROOT/pages/` organization
- Transaction docs separation (`declarative/`, `programmatic/`)
- Meaning of `subs="verbatim,quotes"`

#### 3. DCO (Developer Certificate of Origin)
- Using DCO instead of CLA (since January 2025)
- `git commit -s` for automatic Sign-off
- Required for all commits

#### 4. AsciiDoc include-code Directive (2026-02-22)
Spring uses `include-code::` to reference external code files:
```asciidoc
[source,java,indent=0,subs="verbatim,quotes",role="primary"]
----
include-code::AppConfig[]
----
```

Code files use tag markers:
```java
// tag::snippet[]
@Configuration
@EnableTransactionManagement
public class AppConfig {
    // ...
}
// end::snippet[]
```

#### 5. Spring File Organization Pattern
- Java examples: `framework-docs/src/main/java/org/springframework/docs/{section}/{ClassName}.java`
- Kotlin examples: `framework-docs/src/main/kotlin/org/springframework/docs/{section}/{ClassName}.kt`
- XML examples: `framework-docs/src/main/resources/org/springframework/docs/{section}/{filename}.xml`
- All files include proper copyright headers (2002-present)

#### 6. Documentation Build Process
- Command: `./gradlew :framework-docs:antora`
- Uses Node.js + Antora for documentation generation
- Validates AsciiDoc syntax and callout markers
- Build must succeed before pushing changes
